# Data Dictionary

This document describes the structure and business meaning of the final
analytics tables (data marts) produced by the BigQuery pipeline.

These tables are designed to be consumed directly by BI tools such as Power BI.

---

## Table: `olist_mart.sales_daily`

**Description:**  
Daily sales performance metrics aggregated at the purchase date level.

**Grain:**  
One row per purchase date.

| Column Name | Description |
|------------|------------|
| `purchase_date` | Date when the order was purchased |
| `orders` | Number of distinct orders placed on that date |
| `customers` | Number of distinct customers who placed orders |
| `revenue_items` | Total revenue from item prices (excluding freight) |
| `revenue_freight` | Total freight (shipping) revenue |
| `revenue_gross` | Total revenue including items and freight |
| `aov` | Average Order Value (revenue_gross / orders) |

---

## Table: `olist_mart.category_monthly`

**Description:**  
Monthly revenue and order metrics by product category.

**Grain:**  
One row per product category per month.

| Column Name | Description |
|------------|------------|
| `year_month` | Year and month in YYYY-MM format |
| `category` | Product category name (English when available) |
| `orders` | Number of distinct orders containing products in this category |
| `revenue_gross` | Total gross revenue generated by the category |

---

## Table: `olist_mart.rfm`

**Description:**  
Customer segmentation table based on RFM (Recency, Frequency, Monetary) analysis.

**Grain:**  
One row per customer.

| Column Name | Description |
|------------|------------|
| `customer_id` | Unique identifier of the customer |
| `recency_days` | Number of days since the customer’s last purchase |
| `frequency` | Total number of distinct orders placed by the customer |
| `monetary` | Total gross revenue generated by the customer |
| `r_score` | Recency score (1–5, higher = more recent) |
| `f_score` | Frequency score (1–5, higher = more frequent) |
| `m_score` | Monetary score (1–5, higher = higher spend) |
| `rfm_score` | Concatenated RFM score (e.g. 555, 321) |

---

## Notes

- All monetary values are expressed in the dataset’s original currency.
- These tables are derived from transactional data and are fully reproducible
  using the SQL pipeline provided in the repository.
